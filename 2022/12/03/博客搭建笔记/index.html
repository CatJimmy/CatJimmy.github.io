个人博客搭建——基于Hexo和GitHub

发表于 2022-12-03 更新于 2023-06-14 阅读次数： Valine：
本文字数： 3.3k 阅读时长 ≈ 5 分钟

为什么要搭建个人博客呢？因为很酷！这是直观感受，另外也是需要一个小天地，将所思所想所学从一团蒙昧中抽取、规整，供日后再提取，也可对自己的生命历程回顾时有章可循。

博客搭建的目的：1.笔记本：学习心得；2.日记本：生活体验；3.材料库：书摘文摘

前人栽树，后人乘凉，前辈们的博客搭建教程已经非常完善，但实践中总会出现一些奇奇怪怪的问题。本篇博文主要是记录我搭建博客的心得。

搭建思路

个人博客的搭建需要三件事：服务器、域名 以及博客框架。

1.有了博客框架，我们不再需要手动布置博客的代码，只需要进行个性化设置以及专注于内容即可。常用的博客框架有静态框架如Hexo、Hugo，动态框架如Wordpress。

2.服务器 用来存储我们所有文件，包括博客框架产生的文件以及我们的文章。自己的设备不会24小时开机和联网，所以需要将文件上传到云端。云服务器可购买🐜、🐧等云服务器，也可以使用Github、Gitee等免费仓库作为服务器。

3.而域名则是用来请求访问我们的服务器上文件的网络地址，也可在🐜、🐧、Godaddy等处购买，如果使用Github仓库作为服务器，也可以Github博客地址直接作为我们的域名。

完成了这三件事，博客就部署好啦。下面开始吧！

准备

工欲善其事必先利其器，在正式搭建博客前，先安装两个软件吧。

1.首先是git，类似命令提示符bash，我们用它来上传文件 到云服务器。在git官网下载，安装时选默认设置即可。

安装完成后右键多出两个git选项。

2.第二是node.js，用于安装Hexo，也就是生成博客框架。在node.js官网下载，安装时选默认设置即可。

安装完成后可运行命令提示符检查是否安装成功。

第一步：服务器——GitHub仓库

绑定GitHub

用git上传文件到GitHub需要登陆，每次输入密码十分麻烦，而采用公钥登录就省去了输入密码的步骤。

首先新建一个文件夹，右键打开 Git Bash （注意：git中的复制粘贴不是 Ctrl+C 和 Ctrl+V，而是 Ctrl+insert 和 Shift+insert.）

复制公钥的内容。或者进入C:\Users[你的Windows用户名]\.ssh，找到id_rsa.pub

进入Github主页 - Settings - SSH and GPG keys - New SSH key，将公钥粘贴到key中。

验证是否成功在 Git Bash 中输入

第一次登录输入yes即可。

创建仓库

创建一个仓库名称为[用户名.github.io]，必须使用Github用户名。这它就是我们的服务器啦！
进入仓库后，点击Code，该地址即是仓库地址。

第二步：博客框架——Hexo

安装Hexo

新建文件夹Blog，右键打开Git Bash Here，这个文件夹就是我们本地的博客文件。

生成博客

在浏览器输入http://localhost:4000/ 即可访问，是不是十分简单？

将博客部署到GitHub

但现在只是在本地访问博客，若别人想通过网络访问我们的博客，还需要先将我们的博客文件存放在云端，也就是我们之前搭建的Github仓库。在Blog中用笔记本打开配置文档config.yml，在文件底部Deployment部分填入，注意空格。repo即是我们的仓库地址，branch一般是master，也可能是main。

然后回到 Blog 文件夹中，打开 Git Bash，安装Git部署插件，部署到Github仓库里。

完成以后，打开浏览器，输入 https://[用户名].github.io， 就可以打开你的网页了。

第三步：域名——Freenom

域名可以在阿里云万网，腾讯云等网站购买。在Freenom也可获得免费域名，但免费域名可能不被搜索引擎收录。购买后把我们的博客地址（https://[用户名].github.io）输入即可。

使用

写写博客

打开blog文件夹下的source - _posts，创建.md文件，使用markdown语法进行写作即可！可参考Markdown官方教程，很容易上手。
